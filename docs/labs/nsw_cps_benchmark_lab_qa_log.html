5 QA Decision

The evaluated NSW/CPS benchmark code is ready for training use.

It reproduces the core benchmark logic correctly and exposes method tradeoffs (balance, retention, and benchmark gap) in a way that is consistent with the repository’s design-first style.

Two branches (exact matching and explicit propensity-overlap summaries) are intentional pedagogic extensions beyond a strict minimal NSW/CPS benchmark script and should remain labeled as such.

6 Optional Tightening If Strict Replication Is Needed

  1. Add a “minimal benchmark mode” that keeps only raw, CEM, and entropy balancing branches.
  2. Freeze a single CEM cutpoint specification and report robustness variants separately.
  3. Add a fixed tolerance rule for benchmark closeness (for example, absolute gap under a pre-specified threshold) for pass/fail reporting.
